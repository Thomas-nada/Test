CC Election Voting Process

A Visual Guide to Casting Your Vote

Step 1: View Candidates

Review candidates.

⬇️

Step 2: Select Candidates

Choose up to 7 candidates.

⬇️

Step 3: Submit Vote Intention

Click "Submit your vote".

⬇️

Step 4: Choose Connection Method

Select connection: browser wallet or CLI.

Path 4a: Browser Wallet

4a.1 Sign Vote Metadata

Standard Light Wallets: Sign vote metadata in browser.

Certain Hardware Wallets (e.g., Ledger Nano X/S+, Keystone via Light Wallet):

  • Connect hardware wallet to light wallet.
  • Connect light wallet to voting app.
  • Signal hardware wallet use in app.
  • App prepares vote in transaction metadata.
  • App asks light wallet to sign; light wallet asks hardware wallet to sign; confirm on device.
  • Signature returned to app.
⬇️

4a.2 Receive Vote Proof/Receipt

Receive vote proof/receipt on website.

⬇️

4a.3 Receive Transaction Hash

Receive transaction hash (with vote) once on-chain.

Path 4b: CLI

4b.1 Generate & Obtain Vote Metadata

App generates vote metadata; user downloads/copies it (JSON).

(This path is typically for users of Trezor, older Ledger Nano S models, CLI wallets, Daedalus, or script-based methods.)

⬇️

4b.2 Create & Sign Transaction

Create and sign transaction with metadata using wallet setup.

⬇️

4b.3 Submit Signed Vote File

Submit signed vote file via Ballot CLI API.

⬇️

4b.4 Receive Verifiable Vote Proof/Receipt

Receive verifiable vote proof/receipt for on-chain verification.
